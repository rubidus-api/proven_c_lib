# Third-Party Notices

## Float parse rewrite

The decimal-to-binary64 parsing work under `src/proven/float_decimal.c` and
`src/proven/float_parse.c` is a clean-room implementation for this repository.

- No implementation code was copied or ported from `glibc`, `musl`,
  `MSVCRT`, `BSD libc`, `fast_float`, `double-conversion`, or similar parser
  libraries.
- The design direction follows public algorithm descriptions only:
  - Clinger-style exact fast paths for small decimal values
  - Eisel-Lemire-family fast-path structure and cached-power concepts
  - exact midpoint comparison for the fallback path
- Any cached `5^q` constants currently present in the source are direct exact
  integer values generated by `scripts/generate_float_decimal_tables.py`, not
  imported tables.

The current codebase still uses a bigint exact fallback as the main
correctness path for difficult inputs. Additional fast paths must preserve the
same clean-room rule: use original code and project-local constants only.
