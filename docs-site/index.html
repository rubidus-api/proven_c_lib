Proven C Library

A high-performance C23 systems library with a strict PAL-isolated architecture. Designed for environments requiring zero-cost abstractions, explicit resource control, and mathematically bounded safety.

Build System (nob.c)

`proven` uses a custom build-script-as-code approach (nob.c). It features Command Hashing for incremental builds and a 26-stage TDD pipeline.

1
cc nob.c -o nob
2
./nob debug
3
./nob release -asan

Architectural Principles

  • ✓
    No Floating Point: Core library is fixed-point or integer-based for maximum portability across embedded environments.
  • ✓
    Failure-Atomic: Operations like append_grow either complete fully or revert to original state if OOM occurs.
  • ✓
    PAL-Isolated: Any dependency on OS/libc is wrapped in a proven_sys_... contract.

Multi-core Work Pools (MPMC)

The job system utilizes a Multi-Producer Multi-Consumer (MPMC) Ring Buffer on top of atomic state machines. It is designed to scale across all available physical CPU cores with near-zero scheduling overhead. 

// Thread-safe job submission
proven_job_sys_t *sys;
proven_job_system_init(heap, 4, 1024, &sys);

proven_job_submit(sys, my_heavy_task, arg);
proven_job_system_wait_all(sys);

Stackless Coroutines

Implemented via Duff's Device, coroutines in proven occupy only 4 bytes of persistent state. They enable hundreds of thousands of concurrent tasks without spawning OS threads or incurring stack allocation costs. 

int process_network(struct MyState *s) {
    PROVEN_CORO_BEGIN(&s->coro);
    // ... initial logic ...
    PROVEN_CORO_YIELD(&s->coro);
    // ... resumed logic ...
    PROVEN_CORO_END(&s->coro);
}

Polymorphic Memory Trait

Arena

High-speed bump allocator for short-lived request contexts. O(1) alloc, batch dealloc.

Pool

Fixed-size recycler for objects of the same type. Eliminates fragmentation.

Heap

Standard PAL-isolated system allocation for long-lived application state.

// Creating a generic Vector using a specific allocator
proven_allocator_t arena = proven_arena_allocator(&my_arena);
proven_array_t vec = PROVEN_ARRAY_INIT(arena, int, 100);

// Core primitives (Slices)
proven_mem_view_t read_only = proven_mem_view_from_lit("Hello", 5);
proven_mem_mut_t  writable  = proven_mem_mut_from_buf(buf, 1024);

Open-Addressing Map

"Performance through contiguous layout"

  • Rehash Isolation: Utilizes scratch allocators to manage self-aliased references during array migrations.
  • Tombstone Recovery: Mathematical probing skips deleted slots efficiently without fragmented chaining.

Intrusive Linked List

"Kernel-style zero-allocation links"

Nodes are embedded directly within data structures using the PROVEN_CONTAINER_OF macro, ensuring zero external heap interactions and maximum cache locality. 

PROVEN_LIST_FOR_EACH_SAFE(node, tmp, &my_list) {
    MyData *d = PROVEN_CONTAINER_OF(node, MyData, link);
    // ... atomic processing ...
}

High-Speed MMap Extractors

Proven supports zero-copy file processing by mapping physical files directly into the virtual address space. This allows O(1) file slurping with minimal kernel-context switches. 

proven_result_mmap_t res = proven_mmap_create(handle, 0, size, PROVEN_MMAP_PROT_READ, false);
if (PROVEN_IS_OK(res.err)) {
    proven_u8str_view_t content = proven_mmap_view(res.value);
    // Scan direct virtual memory without read() calls...
}

26 Verify Stages (TDD Pipeline)

Every commitment passes a rigorous 26-stage integrity validation. We stress-test for load-factor boundaries, self-aliasing rehash bugs, and MPMC contention.

Memory Isolation

Validated by Phase 1-6. Verifies bounds slices and PAL deallocation routing logic.

Structure Integrity

Validated by Phase 8-13. Tests Vector growth, Map rehashing, and List pointer math.

Concurrency Stress

Validated by Phase 23. MPMC Job system under heavy contention with ASan/TSan support.
