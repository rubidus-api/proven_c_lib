Proven C Library

An architecture and specification dashboard for the next-generation C library aiming for absolute platform independence, zero-cost interfaces, and perfect memory/thread control.

No-CRT Philosophy

The core logic never directly invokes external standard library APIs like <stdlib.h>. All system calls are completely isolated and routed through the Platform Abstraction Layer (PAL), ensuring 100% consistent execution and memory safety across any target OS.

VTable Polymorphism

Memory allocation is decoupled from concrete implementations via inverted dependencies (DI). All operations reference a proven_allocator_t interface, allowing zero-cost scaling between high-speed Arenas and standard Heap allocators.

Thread Safety Contract

To achieve extreme theoretical efficiency, `proven` entirely excludes internal locks (Mutex, Spinlocks) within its core arrays and data structures. It functions flawlessly as a single-threaded entity where each thread isolates its resources. For concurrent mutation, external synchronization (External Lock) at the application level is strictly mandated.

Stackless Coroutines (Duff's Device)

A zero-allocation, macro-based coroutine engine resolving state jumps exclusively using a 4-byte int without spawning OS threads or allocating execution stacks.

struct Fetcher {
    proven_coro_t coro;
    int progress;
};

int fetch_data(struct Fetcher *f) {
    // Zero context-switching overhead
    PROVEN_CORO_BEGIN(&f->coro);

    proven_println("Loading...");
    PROVEN_CORO_YIELD(&f->coro); // Pauses and returns

    while(f->progress < 100) {
        PROVEN_CORO_YIELD(&f->coro);
    }
    
    PROVEN_CORO_END(&f->coro);
}

Lock-Free MPMC Job System

A high-velocity scheduler running on C11 stdatomic. Enables distributing thousands of heavy payloads to physical cores via Lock-Free Ring Buffers.

// 1. Initialize Thread Pool (Lock-Free)
proven_job_system_init(heap, 4, 1024);

// 2. Submit thousands of jobs without blocking
for (int i = 0; i < 1000; i++) {
    while (!proven_job_submit(my_task, arg)) {
        proven_sys_thread_yield(); // Ring buffer full
    }
}

// 3. Threads automatically consume tasks concurrently.
proven_job_system_shutdown();

Result Pattern vs Out Pointers

Prohibits traditional C patterns like returning `NULL` or modifying output pointers (`out_val`). All errors are strictly wrapped in a Result struct containing both the error reason (`ERR`) and the payload (`VALUE`).

// Deprecated C Standard Approach (Forbidden)
int* p = malloc(size);
if (!p) { ... } // Error reason unknown

// Proven Library Pattern (Safe, Explicit)
proven_result_mem_mut_t res = alloc.alloc_fn(ctx, 128, 8);
if (!proven_is_ok(res.err)) {
    // Exact reason like PROVEN_ERR_NOMEM available for early return
    return res.err; 
}
// Perfectly safe pointer usage
proven_byte_t *safe_ptr = res.value.ptr;

proven_array_t / list_t

Generic Array (Vector): Dynamically growing arrays backed by polymorphic allocators. Even in C, macro wrappers fully infer type parameters to provide type-safe push and pop operations.
Intrusive Linked List: "Zero-allocation" node linkages that do not allocate or free memory themselves. Extracts parent structures mathmetically via PROVEN_CONTAINER_OF. Built for kernel-level optimization.

proven_map_t / ring_t

Open-Addressing Hash Map: A seamless flat router eliminating OOM fragmentation and cache misses caused by Linked List chaining. Equipped with Tombstone states and SplitMix64 hashing, ensuring extreme O(1) speeds.
Circular Ring Buffer: Lock-free/Bounded pipeline engine that rotates indices continuously within a pre-determined capacity buffer.

Filesystem & MMap Engine

Integrates system file slurping, directory traversal (with automatic sorting), permission controls (chmod), and Advisory Locking into the Platform Abstraction Layer (PAL). Crucially, it supports memory-mapped files (mmap) for zero-copy file I/O operations directly mapped into virtual memory. 

proven_fs_handle_t file = proven_sys_fs_open("data.bin", PROVEN_FS_MODE_READ);

// Zero-copy I/O: MMap the file directly into virtual memory.
proven_result_mmap_t mapping = proven_mmap_create(
    file, 0, size, PROVEN_MMAP_PROT_READ, false);

if (proven_is_ok(mapping.err)) {
    proven_u8str_view_t view = proven_mmap_view(mapping.value);
    // Directly process text view without allocating read buffers...
    proven_mmap_close(mapping.value);
}

Formatting (fmt.h)

To eliminate dependencies on classic C standard I/O (<stdio.h>) and mitigate format string vulnerabilities, the library provides a Python/Zig style safe formatter. It supports auto-indexing, padding, and type-safe argument injection.

Variadic Architecture: Functions do not use va_list. Instead, macros like proven_println wrap __VA_ARGS__ into compound literal arrays ((proven_arg_t[]){ ... }), allowing the compiler to deduce the argument count statically using sizeof. This fully avoids runtime counting and missing sentinel errors.

Auto-Flushing: Calls to formatting macros are passed directly to OS-level standard out/error handles. The proven_println variant implicitly attaches a trailing newline character \n and synchronously issues a proven_sysio_flush to the underlying system channel to ensure exact determinism. 

// Output directly through proven_sys_io, bypassing libc
proven_file_t file = proven_sysio_stdout();

// Secure, Zig-style formatting. Automatically flushes to console.
// e.g. prints: "Count: 00100" followed by a newline (\n)
proven_println_to(file, "Count: {:0>5}", proven_arg_i32(100));

Scanning (scan.h)

Safely parses structured input from memory views or standard input. Unlike `scanf`, which can cause buffer overflows, this scanning module operates on strict bounds and securely parses integers, floats, and character sets. 

proven_u8str_view_t input = PROVEN_LIT("ID: 402");
int32_t parsed_id = 0;

// Scan text to extract integer
if (proven_scan_i32(&input, "ID: {}", &parsed_id)) {
    // parsed_id = 402
    // input view boundary is safely advanced automatically
}

TDD Integrity Validation Pipeline (20 Phases)

The proven library is strictly validated by a proprietary build script (nob.c) executing 20 rigorous TDD phases without external frameworks. These tests simulate pointer overruns, forced OOM, memory leaks, and concurrent race conditions to test architectural integrity to its absolute limit.

1. Foundation & Memory Guard

  • Phase 1-3 (Primitives): Evaluates View/Mut slices against Out-Of-Bounds (OOB) exceptions and mathematical alignment behavior.
  • Phase 4 (Arena Allocator): Injects deliberate Out-of-Memory (OOM) triggers across 200+ bytes to ensure graceful PROVEN_ERR_NOMEM returns instead of crashes.
  • Phase 6 (Polymorphic Bindings): Ensures that `free` is effectively a no-op inside the Arena but routes strictly to OS syscalls under dynamic Heap bindings.

2. Zero-Cost Formatting & String

  • Phase 5, 7 (U8 Strings): Validates that the PROVEN_LIT macro compresses array length statically to O(1) without strlen overhead, and strings maintain null seal termination on truncations.
  • Phase 16 (Formatting): Verifies secure Zig-style structural formatters overriding malicious Format String vulnerabilities intrinsic to ``.
  • Phase 18 (System I/O): Connects secure buffers directly to `stdout` file descriptors entirely independent of the CRT.

3. Intensive Data Structures

  • Phase 8-9 (Vectors): Verifies dynamic array buffer swaps and mathematically reverse-engineered Zero-allocation LinkedList via PROVEN_CONTAINER_OF.
  • Phase 10-11 (Map & Ring): Evaluates Open-Addressing HashMaps under heavy 75% load constraints. Verifies Tombstone state recovery and cache-hit optimization during array migration.
  • Phase 12 (Algorithms): Confirms macro-injected `Quicksort` and `Binary Search` preserve mathematically sound O(N log N) thresholds.

4. Low-Level I/O & Concurrency

  • Phase 13-15, 17 (I/O & Mmap): Corroborates cross-platform file slurry maps into virtual memory with strictly Zero-copy semantics.
  • Phase 19 (Coroutines): Tests Duff's Device state machine spanning multiple execution blocks retaining its byte integrity solely in an integer instance context.
  • Phase 20 (Lock-Free Job Pool): Verifies ring buffer endurance against hundreds of physical thread pollutions without Mutex. Ensures 100% job execution delivery under atomic racing conditions.
